# VoltPro Training Portal

Staff training video portal: watch, comment/ask questions, get an AI-generated quiz after
each video, and track who's watched what. Built with Flask + SQLAlchemy, branded to match
voltpro.com.au.

## What's working right now (built and verified locally)

- Login (individual staff accounts, admin-created)
- Video catalog grouped by category, VoltPro-branded (real logo/colors sampled from the
  official logo file, header/nav pattern matching the live site)
- Video player with per-user watch-progress tracking (resumes where you left off)
- Comments / Q&A thread under each video
- Auto-quiz (5 MCQs generated by Claude from the video's transcript) once you've watched 90%+
- Admin dashboard: staff x video matrix (completion %, quiz scores)
- Admin "add staff" page
- Explicit per-video ordering within a category (`Video.sort_order`) instead of relying on
  alphabetical title sort — see "Reordering videos" below
- "Ask the Regs" — staff can ask a plain-English question and get an answer grounded in the
  actual text of AS/NZS 3000:2018 (with page citations), not general AI knowledge. All
  questions are logged and visible to admins — see "Ask the Regs" below

**Content status:** all 16 videos currently in the "Full Domestic Videos" Drive folder are

and Lighting subfolders were empty in Drive at the time, so those categories have no videos

## Running it locally

```
cd ~/voltpro-training-portal
./run.sh
```

`run.sh` runs pending database migrations, seeds default categories/admin account, then
starts the server. Visit http://localhost:5060 — log in with the admin account from `.env`
(`ADMIN_USERNAME` / `ADMIN_PASSWORD`, defaults to `matt` / `training123` — **change this**
before it's ever exposed to the internet).

## Database migrations

Schema changes are tracked with Alembic (via Flask-Migrate), not manual `ALTER TABLE` or
`db.create_all()`. Workflow for any future schema change (new column, new table):

```
source .venv/bin/activate
export FLASK_APP=server.py
# after editing app/models.py:
flask db migrate -m "describe the change"   # autogenerates migrations/versions/<rev>.py
flask db upgrade                             # applies it to your local DB
```

Review the autogenerated migration file before committing — Alembic is usually right but
occasionally needs a manual nudge (e.g. it won't detect a column rename, it'll see it as a
drop+add). `flask db upgrade` is safe to re-run any time (no-op if already current), and is
already chained into `run.sh` and will be part of the Render start command, so both local
dev and production stay in sync automatically as long as migrations are committed to git.

## Reordering videos within a category

Videos sort by `sort_order` (ties broken alphabetically). All default to `0` today, which
falls back to alphabetical — fine until a category needs a deliberate sequence (e.g. a
multi-part series where the part names don't sort correctly as plain text). To set one:

```
python scripts/set_video_order.py "Category Name" \
  "First video title" \
  "Second video title" \
  "Third video title"
```

Any video in the category not listed keeps its current position, sorted after the ones you
did list.

## Adding more videos (ingestion)

Ingesting a video is two steps, because the source files are large (many are 500MB–1.2GB)
and Google Drive's API-based download isn't practical for files that size:

**Step 1 — download the source file.** Open the video in Drive (in a browser where you're
logged into the account it's shared with) and download it to `~/Downloads`. For files over
~35MB, Google shows a "can't scan this file for viruses" interstitial — click "Download
anyway" rather than assuming the direct link downloads immediately.

**Step 2 — convert, store, and register it:**

```
source .venv/bin/activate
python scripts/ingest_video.py \
  --source "~/Downloads/Some Video.wmv" \
  --category "Earthing" \
  --title "Some Video" \
  --drive-id "<the Drive file ID from its URL>"
```

Valid `--category` values: Apprentice On Boarding, Meter Box Upgrade, Earthing, IXL
Installation, Lighting, Testing, 3 Phase, Generator, CCTV, Tesla, Toyota. The script
converts to MP4 (h264/aac), stores it via the configured storage backend, probes its
duration, and creates/updates the `Video` row. Re-running with the same `--drive-id`
updates the existing record rather than duplicating it.

**Step 3 — transcribe and generate its quiz:**

```
python scripts/transcribe_videos.py --video-id <the new video's id>
python scripts/generate_quizzes.py --video-id <the new video's id>
```

(Quizzes also generate automatically the first time a staff member unlocks one — running
`generate_quizzes.py` ahead of time is optional, just avoids the first viewer waiting on it.)

## Transcripts + quizzes

Transcription uses `faster-whisper` locally (free, no API cost) — the `small.en` model ran
at 9x-45x realtime on this Mac's CPU, so the full ~3.5-hour library took about 12 minutes.
Quiz generation calls the Claude API (`app/quiz.py`) from the transcript — both are fully
wired, tested end-to-end with live credentials, and already run for all 16 videos.

## Ask the Regs

Staff-facing tab (`/ask`) where anyone can ask a question about the wiring rules and get an
answer sourced only from the actual standard text — not the AI's general training
knowledge. Built as lightweight retrieval-augmented generation, no vector DB needed at this
corpus size:

1. `scripts/extract_regulations.py` extracts the AS/NZS 3000 PDF page-by-page (via
   poppler's `pdftotext`) into `RegulationChunk` rows — one per printed page, watermarks and
   headers stripped. The source PDF itself is never copied into the project or committed to
   git, only the extracted text, and that text is never exposed to staff directly — only
   AI-synthesized answers with page citations (see the copyright note below).
2. `app/regs.py` does TF-IDF-weighted keyword search over the chunks for a given question,
   then widens to neighboring pages of the top hits — table/figure-heavy pages (like
   Table 5.1 itself) are mostly numbers and score poorly on keyword overlap even when
   they're the actual answer, so pulling in neighbors of strong text matches catches this
   without needing real semantic/embedding search.
3. The matched excerpts + question go to Claude (`answer_question()`), instructed to answer
   *only* from the provided excerpts and say so plainly if they don't contain a clear
   answer, rather than fall back to general knowledge. Verified this behavior directly:
   correctly answers well-covered questions with accurate figures and page citations,
   correctly admits uncertainty when the right clause isn't in the retrieved excerpts, and
   correctly refuses out-of-scope questions ("what's the capital of France?") instead of
   hallucinating an answer.
4. Every question + answer is logged (`RegulationQuestion`) and visible staff-wide on the
   `/ask` page itself (a running FAQ) and to admins at `/admin/questions` — genuinely useful
   signal for what the crew is actually unsure about.

To (re-)populate the index: `python scripts/extract_regulations.py --source "/path/to/AS3000.pdf"`
(safe to re-run — it replaces all chunks). A permanent caveat banner is shown on the `/ask`
page: this is AI-generated guidance, not a compliance ruling, and staff should verify
against the current standard or a supervisor before relying on it for real work.

**Copyright note:** AS/NZS 3000 is a commercially licensed standard, not freely
distributable. Storing extracted text server-side to power internal search for your own
staff (behind login, answers-with-citations rather than raw document access) is a more
defensible use than public redistribution, but isn't a substitute for confirming this is
covered by the terms of your specific license — worth checking if that matters to you.

## Going live — what's left

1. **GitHub** — account created; repo not yet created. Once it exists, code gets pushed
   there and Render deploys from it.
2. **Render** — account created; Web Service + Postgres not yet set up.
3. ~~Cloudflare R2~~ — done, credentials verified working.
4. ~~Anthropic API key~~ — done, quiz generation verified working live.

Once Render's set up: push code to GitHub, connect Render to the repo, add a Postgres
instance (Render provides `DATABASE_URL` — paste into `.env`/Render's env vars), set the
Render start command to run migrations + seed before starting the server (see `run.sh` for
the exact commands), switch `STORAGE_BACKEND` to `r2` and upload the local video library to
the R2 bucket, then point `training.voltpro.com.au` at the Render URL via HostPapa's DNS
zone editor.

Storage costs: the full video library is likely tens of GB; R2 storage runs about
$0.015/GB/month with **no egress fees** (important since staff will re-watch videos —
AWS S3 would charge for every view). Claude API cost per quiz is trivial (one short call
per video, generated once and cached in the database).

## Project layout

```
app/
  __init__.py    — app factory, wires migrations, `flask seed` CLI command
  config.py      — env var loading
  models.py      — SQLAlchemy models
  auth.py        — Flask-Login, admin_required decorator
  routes.py      — catalog, video player, progress API, comments API, quiz
  admin.py       — staff management, tracking dashboard, staff-question log
  quiz.py        — Claude API quiz generation
  ask.py         — Ask the Regs routes (staff Q&A page + API)
  regs.py        — Ask the Regs retrieval (TF-IDF search) + Claude answer synthesis
  storage.py     — local-disk / Cloudflare R2 storage abstraction
  templates/, static/
migrations/       — Alembic migration history (see "Database migrations" above)
scripts/
  ingest_video.py        — convert + store + register a downloaded video
  transcribe_videos.py   — faster-whisper transcription
  generate_quizzes.py    — pre-generate quizzes via Claude
  set_video_order.py     — set explicit display order within a category
  extract_regulations.py — extract AS/NZS 3000 PDF into RegulationChunk rows
storage/videos/   — local video files (dev only; not used once STORAGE_BACKEND=r2)
```
